🔧 Heritage Method Implementation

Core Heritage Algorithm

' Heritage file creation - preserves design relationships Sub CreateHeritageFile(originalPath, newPath, groupName) ' Open original part Set partDoc = invApp.Documents.Open(originalPath, False) ' Create heritage copy with new name partDoc.SaveAs newPath, True ' True = create heritage link ' Close original (heritage copy remains) partDoc.Close LogMessage "HERITAGE: Created " & newPath & " from " & originalPath End Sub ' Assembly reference updating Sub UpdateAssemblyReferences(assemblyDoc, originalPath, newPath) For Each occ In assemblyDoc.ComponentDefinition.Occurrences If occ.ReferencedFileDescriptor.FullFileName = originalPath Then ' Replace reference to point to heritage file occ.Replace newPath, True LogMessage "ASSEMBLY: Updated reference " & originalPath & " → " & newPath End If Next End Sub

Base View Detection Algorithm

' Automatic base view detection and title formatting Function DetectBaseView(view) ' Check if view is a base view (not projected, section, or detail) If view.ViewType = kBaseDrawingViewType Then DetectBaseView = True ' Additional validation logic here Else DetectBaseView = False End If End Function ' Title formatting with quantity detection Sub FormatTitleBlock(baseView, assemblyDoc) ' Get referenced model name modelName = GetModelName(baseView) ' Detect if part or assembly If IsAssembly(baseView) Then quantity = CalculateQuantity(assemblyDoc, modelName) title = modelName & vbCrLf & quantity & "-OFF REQ'D" & vbCrLf & "SCALE " & GetScale(baseView) Else title = modelName & vbCrLf & "SCALE " & GetScale(baseView) End If ' Apply formatting with font sizes ApplyTitleFormatting(baseView, title) End Sub

  • Format: Pipe-separated values with full file paths

🗄️ Registry Database Schema

Windows Registry Structure

Location: HKEY_CURRENT_USER\Software\InventorRenamer\

Key Naming Convention:

[PREFIX][GROUP_CODE] = [COUNTER_VALUE] Examples: PLANT1-000-PL = 15 (Platework counter for PLANT1-000- prefix) PLANT1-000-B = 8 (Beam counter for PLANT1-000- prefix) NCRH01-000-CH = 3 (Channel counter for NCRH01-000- prefix)

Counter Management:

  • Type: REG_DWORD (32-bit integer)
  • Range: 1 to 999999
  • Increment: +1 per part processed
  • Persistence: Survives reboots and script executions

⚠️ Error Handling

Error Recovery Patterns

' File access error handling On Error Resume Next Set partDoc = invApp.Documents.Open(partPath, False) If Err.Number <> 0 Then LogMessage "ERROR: Cannot open " & partPath & " - " & Err.Description Err.Clear On Error GoTo 0 Exit Sub End If On Error GoTo 0 ' Registry access with fallback On Error Resume Next currentValue = shell.RegRead(regPath & keyName) If Err.Number <> 0 Then currentValue = 0 ' Start at 1 (will be incremented) Err.Clear End If On Error GoTo 0

🔧 System Requirements

Software Dependencies

  • Autodesk Inventor: 2018 or later (tested through 2024)
  • Windows: 10/11 with VBScript support
  • Permissions: Registry write access, file system write access
  • Memory: 8GB+ RAM recommended for large assemblies
  • Storage: 2x project size free space for heritage files

Network Considerations

⚠️ Performance Impact: Network drives add 50-100% processing time. Copy project to local drive for optimal performance.
