CLIENT PROGRESS REPORT - Sheet Metal Conversion Automation
==========================================================

DATE: January 6, 2026
PROJECT: Automated Plate Part Sheet Metal Conversion
STATUS: Critical Issue Resolved - Partial Completion


THE PROBLEM
-----------

When converting plate parts (like underpan components) from standard parts to sheet metal, the flat pattern was displaying incorrectly:

- Instead of showing the large flat plate surface (e.g., 2500mm x 4700mm)
- The flat pattern was rotated 90 degrees showing the thin edge (3850mm x 6mm)
- This made the flat patterns unusable for manufacturing/nesting purposes

Root cause: The Inventor API was selecting the wrong face as the "base face" for the flat pattern. It was picking a small edge face (23,100 mm² area) instead of the large top/bottom face (6,174,892 mm² area).


ATTEMPTS TO FIX VIA STANDARD API
---------------------------------

We tested multiple Inventor API methods to programmatically control which face becomes the base:

1. BaseFace property - Cannot be set, it's read-only
2. CreateDefinition with face parameter - Returns empty/null on these derived parts  
3. Unfold(face) method - Doesn't accept a face parameter
4. FlipBaseFace method - Executes but has no effect
5. Pre-selecting the face - The command ignores pre-selection and waits for user click

All standard API approaches failed. The API simply doesn't support programmatic base face selection during the conversion process.


THE SOLUTION
------------

We found a working solution that combines API automation with keyboard automation:

STEP 1: Calculate which face is the largest (the correct top/bottom face)
STEP 2: Add that face to the selection set in Inventor
STEP 3: Execute the "Convert to Sheet Metal" command
STEP 4: Use SendKeys to automatically press Enter twice:
        - First Enter confirms the pre-selected face
        - Second Enter accepts the default sheet metal settings
STEP 5: Create the flat pattern

This approach works because:
- The command accepts the face that's already in the selection set when Enter is pressed
- SendKeys simulates the user pressing Enter at the right moment
- The result is the correct face is used as the base face


RESULTS
-------

BEFORE FIX:
- Flat Pattern: 3,850mm x 6mm (wrong - showing edge)
- Unusable for manufacturing

AFTER FIX:  
- Flat Pattern: 4,727.9mm x 2,500.7mm (correct - showing large face)
- Proper orientation for DXF export and nesting

TECHNICAL LIMITATIONS
---------------------

1. Requires Inventor window to be visible/active during conversion
   - SendKeys needs to send keyboard input to Inventor
   - Cannot run completely in background

2. User should not touch keyboard while script is running
   - SendKeys types Enter at specific moments
   - Keyboard input could interfere

3. Timing is important
   - Script waits for dialogs to appear before pressing Enter
   - Works reliably but requires the computer to respond at expected speed

RECOMMENDATION

The remaining work is integration - taking the proven single-part conversion and applying it to batch process all plate parts in an assembly. This is straightforward implementation work without the research/trial-and-error required to solve the orientation issue.

TECHNICAL NOTE
--------------

This issue revealed a gap in Autodesk Inventor's API. The programmatic interface for creating sheet metal parts with specific base face selection is incomplete for derived parts. Our solution uses UI automation (SendKeys) as a workaround. This is a legitimate approach used in many automation scenarios where APIs are incomplete. Microsoft Office automation, CAD scripting, and test automation frequently use similar techniques.
